Key Concepts

Arrival Time (AT)

The time at which a process enters the ready queue.

Burst Time (BT)

The total time required by a process for CPU execution.

Completion Time (CT)

The time at which a process completes its execution.

Turnaround Time (TAT)

Total time from arrival to completion. TAT = CT - AT

Waiting Time (WT)

Time spent waiting in ready queue. WT = TAT - BT

Response Time (RT)

Time from arrival until first CPU allocation.

Non-Preemptive

First Come First Serve (FCFS)

How FCFS Works

FCFS is the simplest CPU scheduling algorithm. Processes are executed in the exact order they arrive in the ready queue. It's like a queue at a ticket counter - first person in line gets served first.

Algorithm Steps

  1. Sort all processes by their arrival time
  2. Execute the first process until completion
  3. Move to the next process in the queue
  4. Repeat until all processes are complete

Characteristics

Simple to implement
No starvation
Convoy effect possible
High average waiting time

Formulas Used

Completion Time (CT) = Previous CT + Burst Time Turnaround Time (TAT) = CT - Arrival Time Waiting Time (WT) = TAT - Burst Time

Optimal

Shortest Job First (SJF)

How SJF Works

SJF selects the process with the smallest burst time for execution next. It provides the minimum average waiting time for a given set of processes. This version implements non-preemptive SJF.

Algorithm Steps

  1. Wait for processes to arrive in ready queue
  2. Select the process with shortest burst time
  3. Execute the selected process to completion
  4. Repeat until all processes complete

Characteristics

Minimum avg waiting time
Optimal for batch systems
Starvation possible
Burst time prediction needed

Priority Based

Priority Scheduling

How Priority Scheduling Works

Each process is assigned a priority value. The CPU is allocated to the process with the highest priority. Lower number typically means higher priority (configurable). Processes with equal priority are scheduled using FCFS.

Algorithm Steps

  1. Assign priorities to all processes
  2. Select process with highest priority from ready queue
  3. Execute the selected process
  4. Repeat until all processes complete

Characteristics

Important tasks first
Flexible scheduling
Starvation possible
Aging can prevent starvation

Preemptive

Round Robin (RR)

How Round Robin Works

Round Robin allocates a fixed time slice (quantum) to each process in circular order. If a process doesn't complete within its quantum, it's preempted and placed at the end of the ready queue. This ensures fair CPU time distribution.

Algorithm Steps

  1. Set a fixed time quantum
  2. Add arriving processes to ready queue
  3. Execute process for time quantum or until completion
  4. If not complete, move to end of queue
  5. Repeat until all processes complete

Characteristics

Fair time distribution
Good response time
No starvation
Context switch overhead

Time Quantum Impact

Large Quantum: Behaves like FCFS
Small Quantum: More context switches, better response
Optimal: Depends on average burst time
